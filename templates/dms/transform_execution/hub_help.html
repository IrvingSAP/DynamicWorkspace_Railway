Ayuda — Ejecutar transformación

Dry run (preview) y ejecución completa sobre un archivo de producción.

← Volver a ejecutar

Propósito

Aplica el pipeline publicado (origen → mapeo → reglas → destino) al archivo subido. Requiere versión publicada y un job en estado cargado (producción).

Prerrequisitos

RequisitoDónde se resuelve
Versión publicadaHub del proyecto → Publicar.
Archivo de producciónMódulo Archivo (file intake).
Permiso de ejecuciónMembresía del proyecto (PA / roles habilitados).

Acciones principales

AcciónEfecto
Solo preview / Dry runTransforma un número limitado de filas para validar sin cerrar el job completo.
Ejecutar transformaciónJob completo: genera archivo salida, informe y (si aplica) errores.
DescargasEnlaces al output / report / errors (TTL típico 7 días).
Subir archivoVa a File Intake para cargar producción.
HistorialLista jobs anteriores y descargas vigentes.
